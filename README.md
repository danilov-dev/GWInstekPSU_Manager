# GWInstekPSUManager

Приложение для управления программируемыми источниками питания GW Instek через COM-порт.

## Описание

GWInstekPSUManager — это WPF-приложение, разработанное для удобного управления лабораторными источниками питания GW Instek. Приложение предоставляет графический интерфейс для настройки параметров каналов, мониторинга состояния устройства и управления выходными сигналами.

## Возможности

- **Автоматическое обнаружение COM-портов** — поиск доступных портов и определение устройств GW Instek
- **Управление каналами** — настройка напряжения и тока для каждого канала
- **Мониторинг в реальном времени** — отображение текущих значений напряжения, тока и статуса канала
- **Поддержка нескольких устройств** — возможность подключения и работы с несколькими источниками питания одновременно
- **Гибкие настройки соединения** — настройка baud rate, data bits и других параметров COM-порта
- **OVP/OCP защита** — установка пределов перенапряжения и перегрузки по току
- **Включение/выключение выходов** — индивидуальное управление выходами каналов

## Архитектура проекта

Проект состоит из трёх основных модулей:

### GWInstekPSUManager.Core
- Интерфейсы для сервисов, устройств и каналов
- Модели данных (DeviceInfo, SerialPortInfo, MeasureResponse и др.)
- Базовые события и исключения

### GWInstekPSUManager.Infrastructure
- Реализация сервисов для работы с COM-портами
- Сервисы подключения и управления устройствами
- Протокол обмена данными с устройствами GW Instek
- Сервисы опроса каналов

### GWInstekPSUManager (WPF Application)
- Пользовательский интерфейс на WPF
- ViewModel для MVVM-архитектуры
- Конвертеры и стили для представления данных
- Навигация между экранами

## Технологии

- **.NET 8.0**
- **WPF** (Windows Presentation Foundation)
- **CommunityToolkit.Mvvm** — библиотека для реализации паттерна MVVM
- **Microsoft.Extensions.Hosting** — внедрение зависимостей и хостинг
- **System.IO.Ports** — работа с последовательными портами
- **Extended.Wpf.Toolkit** — дополнительные WPF контролы

## Требования

- ОС: Windows 10/11
- .NET 8.0 Runtime или SDK
- Наличие COM-порта для подключения устройства

## Установка и запуск

### Клонирование репозитория

```bash
git clone <repository-url>
cd GWInstekPSUManager
```

### Сборка проекта

```bash
dotnet restore
dotnet build
```

### Запуск приложения

```bash
dotnet run --project GWInstekPSUManager/GWInstekPSUManager.csproj
```

Или откройте решение `GWInstekPSUManager.sln` в Visual Studio 2022 и запустите проект.

## Использование

1. **Подключение устройства**
   - Подключите источник питания GW Instek к компьютеру через USB/COM
   - Запустите приложение
   - Выберите обнаруженный COM-порт из списка
   - Нажмите кнопку **Connect**

2. **Настройка параметров канала**
   - Перейдите на вкладку устройства
   - Установите требуемые значения напряжения и тока
   - Включите выход канала

3. **Мониторинг**
   - Наблюдайте за текущими показаниями в реальном времени
   - Отслеживайте статус каналов и ошибки

4. **Управление несколькими устройствами**
   - Подключите несколько источников питания
   - Переключайтесь между устройствами в списке подключенных

## Поддерживаемые устройства

Приложение поддерживает источники питания GW Instek серий:
- GPS серии (GPS-3303, GPS-4303 и др.)
- PSP серии
- PEL серии (электронные нагрузки)
- Другие устройства с совместимым SCPI-подобным протоколом

```
GWInstekPSUManager.sln

│   └── HostBuilders/                # Настройка DI контейнера
├── GWInstekPSUManager.Core/         # Базовые интерфейсы и модели
│   ├── Interfaces/                  # Контракты сервисов
│   ├── Models/                      # Модели данных
│   ├── Events/                      # События
│   └── Exceptions/                  # Исключения
└── GWInstekPSUManager.Infrastructure/ # Реализация сервисов
    ├── Services/
    │   ├── ComPortServices/         # Работа с COM-портами
    │   ├── DeviceServices/          # Управление устройствами
    │   ├── ChannelServices/         # Управление каналами
    │   └── ConectionServices/       # Стратегии подключения
```

## Конфигурация

Параметры подключения по умолчанию:
- **Baud Rate**: 115200
- **Data Bits**: 8
- **Parity**: None
- **Stop Bits**: One
- **Read Timeout**: 10000 мс

Эти параметры можно изменить в интерфейсе приложения перед подключением.
